4 Welcome to matplotlib-cpp, possibly the simplest C++ plotting library.

26 A more comprehensive example:
27 ```cpp
28 #include "matplotlibcpp.h"
29 #include <cmath>
30 
31 namespace plt = matplotlibcpp;
32 
33 int main()
34 {
35  // Prepare data.
36  int n = 5000;
37  std::vector<double> x(n), y(n), z(n), w(n,2);
38  for(int i=0; i<n; ++i) {
39  x.at(i) = i*i;
40  y.at(i) = sin(2*M_PI*i/360.0);
41  z.at(i) = log(i);
42  }
43 
44  // Set the size of output image to 1200x780 pixels
45  plt::figure_size(1200, 780);
46  // Plot line from given x and y data. Color is selected automatically.
47  plt::plot(x, y);
48  // Plot a red dashed line from given x and y data.
49  plt::plot(x, w,"r--");
50  // Plot a line whose name will show up as "log(x)" in the legend.
51  plt::named_plot("log(x)", x, z);
52  // Set x-axis to interval [0,1000000]
53  plt::xlim(0, 1000*1000);
54  // Add graph title
55  plt::title("Sample figure");
56  // Enable legend.
57  plt::legend();
58  // Save the image (file format is determined by the extension)
59  plt::save("./basic.png");
60 }
61 ```
62  g++ basic.cpp -I/usr/include/python2.7 -lpython2.7

68 Alternatively, matplotlib-cpp also supports some C++11-powered syntactic sugar:
69 ```cpp
70 #include <cmath>
71 #include "matplotlibcpp.h"
72 
73 using namespace std;
74 namespace plt = matplotlibcpp;
75 
76 int main()
77 {
78  // Prepare data.
79  int n = 5000; // number of data points
80  vector<double> x(n),y(n);
81  for(int i=0; i<n; ++i) {
82  double t = 2*M_PI*i/n;
83  x.at(i) = 16*sin(t)*sin(t)*sin(t);
84  y.at(i) = 13*cos(t) - 5*cos(2*t) - 2*cos(3*t) - cos(4*t);
85  }
86 
87  // plot() takes an arbitrary number of (x,y,format)-triples.
88  // x must be iterable (that is, anything providing begin(x) and end(x)),
89  // y must either be callable (providing operator() const) or iterable.
90  plt::plot(x, y, "r-", x, [](double d) { return 12.5+abs(sin(d)); }, "k-");
91 
92 
93  // show plots
94  plt::show();
95 }
96 ```

192 Installation
193 ------------
194 
195 matplotlib-cpp works by wrapping the popular python plotting library matplotlib. (matplotlib.org)
196 This means you have to have a working python installation, including development headers.
197 On Ubuntu:
198 
199  sudo apt-get install python-matplotlib python-numpy python2.7-dev
200 
201 If, for some reason, you're unable to get a working installation of numpy on your system,
202 you can define the macro `WITHOUT_NUMPY` before including the header file to erase this
203 dependency.
204 
205 The C++-part of the library consists of the single header file `matplotlibcpp.h` which can be placed
206 anywhere.
207 
208 Since a python interpreter is opened internally, it is necessary to link against `libpython` in order
209 to user matplotlib-cpp. Most versions should work, although `libpython2.7` and `libpython3.6` are
210 probably the most regularly testedr.

219 
220 It's easy to use cmake official [docs](https://cmake.org/cmake/help/git-stage/module/FindPython2.html#module:FindPython2) to find Python 2(or 3) interpreter, compiler and development environment (include directories and libraries).

223 
224 ```cmake

226 target_include_directories(myproject PRIVATE ${Python2_INCLUDE_DIRS} ${Python2_NumPy_INCLUDE_DIRS})
227 target_link_libraries(myproject Python2::Python Python2::NumPy)
228 ```
229 
230 **Alternative way (for CMake <= 3.11):**
231 
232 ```cmake

234 target_include_directories(myproject PRIVATE ${PYTHON_INCLUDE_DIRS})
235 target_link_libraries(myproject ${PYTHON_LIBRARIES})
236 ```

274 Why?
275 ----
276 I initially started this library during my diploma thesis. The usual approach of
277 writing data from the c++ algorithm to a file and afterwards parsing and plotting
278 it in python using matplotlib proved insufficient: Keeping the algorithm
279 and plotting code in sync requires a lot of effort when the C++ code frequently and substantially
280 changes. Additionally, the python yaml parser was not able to cope with files that
281 exceed a few hundred megabytes in size.
282 
283 Therefore, I was looking for a C++ plotting library that was extremely easy to use
284 and to add into an existing codebase, preferably header-only. When I found
285 none, I decided to write one myself, which is basically a C++ wrapper around
286 matplotlib. As you can see from the above examples, plotting data and saving it
287 to an image file can be done as few as two lines of code.
288 
289 The general approach of providing a simple C++ API for utilizing python code
290 was later generalized and extracted into a separate, more powerful
291 library in another project of mine, [wrappy](http://www.github.com/lava/wrappy).

294 Todo/Issues/Wishlist
295 --------------------
296 * This library is not thread safe. Protect all concurrent access with a mutex.
297  Sadly, this is not easy to fix since it is not caused by the library itself but
298  by the python interpreter, which is itself not thread-safe.
299 
300 * It would be nice to have a more object-oriented design with a Plot class which would allow
301  multiple independent plots per program.
302 
303 * Right now, only a small subset of matplotlibs functionality is exposed. Stuff like xlabel()/ylabel() etc. should
304  be easy to add.

306 * If you use Anaconda on Windows, you might need to set PYTHONHOME to Anaconda home directory and QT_QPA_PLATFORM_PLUGIN_PATH to %PYTHONHOME%Library/plugins/platforms. The latter is for especially when you get the error which says 'This application failed to start because it could not find or load the Qt platform plugin "windows"
307 in "".'

There is Qt4Agg and GTKAgg and as a back-end is not the default. Set the back end of macosx that is differ compare with other windows or linux os.
